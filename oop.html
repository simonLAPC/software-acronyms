OOP

Object-Oriented Programming

Bundle related data and logic into 'Objects' that are self-encapsulated.

  • Encapsulation: Relvant data and logic live together.

  • Inheritance: Enables code reuse and extension.

  • Polymorphism: Allows objects to share interfaces.

With OOP

Show Code 

# Each department's logic is encapsulated in its own class
class SceneCleaner:
    def clean(self): raise NotImplementedError

class AnimationCleaner(SceneCleaner):
    def clean(self):
        print('Deleting lights and FX nodes...')

class LightingCleaner(SceneCleaner):
    def clean(self):
        print('Deleting animation controls and constraints...')
        
# Easy to add a new cleaner without touching existing code

# Easy to use and extend
pipeline_steps = {
    'Animation': AnimationCleaner(),
    'Lighting': LightingCleaner(),
}

department_to_clean = 'Lighting'
pipeline_steps[department_to_clean].clean()
