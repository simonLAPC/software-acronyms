OOP

Object-Oriented Programming

A programming paradigm based on "objects", which bundle related data and behaviors. It helps structure programs into reusable, modular components.

  • Encapsulation: Bundles data and methods together.

  • Inheritance: Enables code reuse and extension.

  • Polymorphism: Allows objects to share interfaces.

Without OOP (Procedural)

Show Code 

# Hard to maintain and extend
def handle_enemy_attack(enemy_type, health):
    if enemy_type == 'goblin':
        print('The goblin attacks with its rusty dagger!')
        health -= 5
    elif enemy_type == 'dragon':
        print('The dragon breathes a torrent of fire!')
        health -= 25
    elif enemy_type == 'orc':
        print('The orc swings its massive club!')
        health -= 15
    # Adding a new enemy requires modifying this function...
    return health

player_health = 100
player_health = handle_enemy_attack('dragon', player_health)
print(f'Player health: {player_health}')

With OOP

Show Code 

# Clean, modular, and extensible classes
class Enemy:
    def attack(self): raise NotImplementedError

class Goblin(Enemy):
    def attack(self):
        print('The goblin attacks with its rusty dagger!')
        return 5

class Dragon(Enemy):
    def attack(self):
        print('The dragon breathes a torrent of fire!')
        return 25

# Easy to use and extend
player_health = 100
enemies = [Goblin(), Dragon()]

for enemy in enemies:
    player_health -= enemy.attack()

print(f'Player health: {player_health}')
